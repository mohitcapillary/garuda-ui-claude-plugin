Run the **test-writer** agent to generate unit tests for a feature after `/hld-to-code` has produced the organism code.

## Instructions

1. Read and follow `.claude/agents/test-writer.md`.
2. If `<feature-slug>` is not provided, ask the user for it.
3. Verify `claudeOutput/build/<feature>/build-log.jsonl` exists (written by `/hld-to-code`). If not, halt — the test writer needs to know which organisms to test.
4. Execute the 5 steps (read context → create tests dir → write 3 batches → write files → write log).
5. Halt on ambiguity via `AskUserQuestion`. Do NOT invent test cases if the generated source doesn't declare the needed exports/actions.

## Examples

```bash
# Generate all three batches (reducer, saga, component)
/write-tests benefits-listing

# Generate only Redux tests (reducer + saga) — skip Component tests
/write-tests benefits-listing --batch redux

# Generate only Component tests
/write-tests benefits-listing --batch component

# Regenerate tests to fill specific coverage gaps (from /test-eval output)
/write-tests benefits-listing --gaps claudeOutput/tests/benefits-listing/gaps.json
```

## What it does (3 batches, halts between on failure)

| Batch | File | Coverage |
|---|---|---|
| 1 | `reducer.test.js` | Every switch case, initial state, unknown action (100% branch) |
| 2 | `saga.test.js` | Every worker × 3 paths (success/failure/error) with `expectSaga` + `matchers.call.fn` (100% worker) |
| 3 | `Component.test.js` | Render + loading + data + error + empty states + every interaction handler (≥80% component) |

## Non-negotiable rules (per `shared-rules.md`)

- Import from `app/utils/test-utils.js` — NEVER `@testing-library/react` directly
- Mock `utils/bugsnag` in every file
- No Enzyme (RTL only)
- Use `renderWithProvider` / `renderWithRouter` / `expectSaga` / `matchers.call.fn`

## Prerequisites

- `claudeOutput/build/<feature>/build-log.jsonl` exists
- `claudeOutput/hld/hld-<feature>.md` exists
- Target garuda-ui has `app/utils/test-utils.js` available
- Jest + `redux-saga-test-plan` installed in target

## Outputs

| Artifact | Location |
|---|---|
| Test files | `<GARUDA_UI_PATH>/app/components/organisms/<Name>/tests/*.test.js` |
| Write log | `claudeOutput/tests/<feature>/test-write-log.jsonl` |

## Follow-up

After `/write-tests` completes, run `/test-eval <feature>` to run Jest and verify coverage meets the threshold in `skills/config.md`.

## When NOT to use

- Before `/hld-to-code` runs (no code to test)
- For integration tests — use a separate runner (MSW+handlers setup is documented in `skills/coding-dna/testing/ref-mocking.md` but not auto-generated by this agent)
